# ACADIA Reference Indexer — Google Apps Script web app

A web app that lets each ACADIA author upload their final paper (`.docx`), auto-extracts the
title, authors, and references, lets the author correct anything flagged, and appends
submission-ready rows to two tabs of a Google Sheet (`title_authors`, `references`) that
editors export to `.xlsx` for CumInCAD. It moves reference indexing — historically the
months-long bottleneck — to the authors, while automating the hard part (parsing).

It lives entirely in Google: the script is **bound to one Google Sheet**, uploads are stored
in an **auto-created Drive folder**, the data is in the Sheet. No server, no IDs to manage,
**no code to edit** — all configuration is plain cells in a **Settings** tab.

---

## Setup (one time, ~10 minutes, no coding)

You do this once. After that, every future year is just editing a few cells.

### 1 · Create the Sheet
Open **[sheets.new](https://sheets.new)** (a blank Google Sheet). Name it, e.g.,
"ACADIA 2026 Indexing".

### 2 · Add the tool's code (paste 2 files)
In the Sheet's top menu: **Extensions → Apps Script**. A script editor opens in a new tab.
- In the default `Code.gs`, select all and replace it with the contents of **`dist/Code.gs`**.
- Click **＋ → HTML**, name the file exactly **`Index`**, and paste the contents of **`dist/Index.html`**.
- Click **Save** 💾.

> (`dist/` is generated by `build.py` from the modular source files; see *Developing* below.)

### 3 · Run "First-time setup" (one click)
Go back to the Sheet tab and **reload the page**. A new **ACADIA Indexer** menu appears.
Click **ACADIA Indexer → 1. First-time setup** and approve the permission prompt (it's your
own script). This creates the `title_authors`, `references`, and **Settings** tabs, a
**How to publish** tab, and the uploads folder — automatically. No IDs to copy anywhere.

### 4 · Fill in the Settings tab
On the **Settings** tab, edit **column B**. The important ones are the three
**"Volume … SOURCE"** rows — paste the **exact CumInCAD wording** for each volume (replace
the `<…>` placeholders). Everything else has sensible defaults.

### 5 · Publish and get the author link
Click **ACADIA Indexer → 2. How to publish** — it lays out the exact clicks:
**Extensions → Apps Script → Deploy → New deployment → Web app**, *Execute as:* **Me**,
*Who has access:* **Anyone**, **Deploy**, then copy the **Web app URL**. Send that URL to authors.

### Future years — no coding, no re-deploy
Reuse the same Sheet: clear last year's rows from `title_authors`/`references`, update the
three SOURCE rows on the **Settings** tab, and reuse the same link. (Or *File → Make a copy*
of the whole Sheet — the tool travels with it — and run First-time setup again.)

## For authors (one-screen instructions to circulate)

1. In Word, make sure your references use the **ACADIA-Reference** paragraph style (and your
   title/authors/keywords use the ACADIA template styles). That's what makes the import clean.
2. Open the indexer link. Choose your **track**, type your **paper number** (from your
   acceptance email), enter your **contact email**, and upload your **`.docx`**.
3. Review the parsed references. Orange-highlighted rows need a look (e.g. an unusual year).
   Fix anything wrong inline — especially author **First/Last** (middle names go in *First*).
4. Click **Confirm & submit**. Done. (Re-submitting the same paper number safely replaces it.)

## For editors (after submissions arrive)

- The two tabs are already in CumInCAD's exact column order. To submit: **File → Download →
  Microsoft Excel (.xlsx)** for each tab.
- Each `title_authors` row's source `.docx` is in the auto-created uploads folder, named
  `‹ID›__‹original›.docx`.
- **ACADIA Indexer → Run self-test** logs `ALL PASSED` if the parser is healthy.

## How it works

```
Author opens the link → picks track + types paper number (→ ID acadia26_v{vol}_{NN}),
                        enters email, uploads .docx
   └─ processUpload(): saves the .docx to Drive, unzips it, reads word/document.xml,
        extracts by paragraph STYLE (ACADIA-Title / ACADIA-Author / ACADIA-Reference),
        parses each reference into authors / year / title / source (year-anchored Chicago)
   └─ Confirmation screen: author fixes any flagged row, edits First/Last, adds/removes rows
   └─ commitSubmission(): appends 1 title_authors row + N references rows (locked, idempotent)
Editors later: File → Download → .xlsx → submit to CumInCAD
```

**Why it's reliable:** references are read by the `ACADIA-Reference` paragraph style (clean,
one per entry) rather than scraped from a PDF; and the parser anchors on the **year** so it
handles both Chicago variants (author-date *and* notes-bibliography), ALL-CAPS, initials,
smart quotes/UTF-8, year ranges, and missing years — flagging anything uncertain for the
author instead of silently guessing.

## Files

| File | Role |
|---|---|
| `Setup.gs` | The **ACADIA Indexer** menu, one-click `setup()`, auto Settings tab + uploads folder. |
| `Config.gs` | Defaults + `getConfig()` (reads the in-sheet Settings tab) + ID/SOURCE helpers. |
| `Code.gs` | `doGet`, endpoints (`processUpload`/`commitSubmission`/`getConfigForClient`), `runTests()`. |
| `Docx.gs` | Unzip + XmlService walk → title/authors/keywords/references by style. |
| `RefParser.gs` | `parseReference()` — splits one Chicago reference into authors/year/title/source + flags. |
| `NameSplit.gs` | `splitAuthorBlock()` — author display name → First/Last pairs. |
| `SheetIO.gs` | Sheet writes (lock + delete-then-append idempotency, exact headers), Drive save. |
| `Index/Stylesheet/JavaScript.html` | The two-stage UI (upload → confirm). |
| `dist/` | Built paste bundle: `Code.gs` (all .gs combined) + `Index.html` (CSS+JS inlined). |
| `appsscript.json` | Manifest (V8, scopes). Optional to paste — the Deploy dialog auto-detects scopes. |

Parsing logic is mirrored and regression-tested offline in [`../prototype/`](../prototype)
(`python test_acadia.py`, 31 checks); `Code.gs` `runTests()` re-runs the key checks in Apps Script.

## Developing

Edit the **modular** files (the `.gs` and the three `.html`), then rebuild the paste bundle:

```bash
cd apps-script
python build.py               # → dist/Code.gs, dist/Index.html, demo.html
python -m http.server 8077    # open http://127.0.0.1:8077/demo.html for a mock-backend preview
```

The demo uses an in-page mock for `google.script.run`, so the UI is clickable without
deploying (upload any `.docx`; the mock returns sample parsed data).

Prefer the CLI? You can also push the modular files with
[`clasp`](https://github.com/google/clasp) (`clasp create --type sheets` against the bound
project, then `clasp push`) — but the paste-2-files flow above needs no tooling.

## Notes & limits
- Apps Script can't accept a raw file over `google.script.run`, so the browser sends the
  `.docx` as base64 (papers are a few MB — well within limits).
- If an author didn't apply the `ACADIA-Reference` style, the app falls back to scanning for a
  "References" heading and flags the result — but styled references are far more reliable.
- The author confirmation step is the safety net: uncertain parses are flagged, never silently
  guessed.
